# 微信 GUI Agent 项目目录结构说明

随着功能日益丰富，为了支撑后续的**图形配置界面(UI)**、**人设库**、**联系人配置库**和**RAG本地知识库**，项目已从单文件平铺重构为标准的前后端解耦架构。特此记录最新目录结构，便于后续开发与查找。

## 目录结构全局纵览

```text
/e:/win_WeChat_AI
├── core/                   # 🧠 核心自动化主引擎引擎模块区 (Backend)
│   ├── __init__.py         # 使 core 成为标准 Python 包
│   ├── engine.py           # 原 main.py，核心引擎类 (WeChatEngine)，被外部启动/干预
│   ├── vision.py           # 📸 屏幕截取、HSV特征提取、坐标识别引擎
│   ├── ocr_parser.py       # 🔍 PaddleOCR解析、文字提取、联系人识别、短文本哈希去重
│   ├── agent.py            # 🤖 AI大脑模块，负责组装人设 Prompt 与调用大语言模型 API
│   ├── action.py           # 🖱️ 物理执行层，强接管键盘鼠标，控制剪贴板和 Ctrl+V
│   ├── window_manager.py   # 🪟 Windows 底层窗口管理器，拉起微信防遮挡、控制置顶等
│   └── anti_risk.py        # 🛡️ 模拟人类行为防风控模块（错别字替换、时延加入）
│
├── data/                   # 🗄️ 数据与持久化配置区
│   ├── config.yaml         # 系统的核心基础坐标配置、大模型 API 参数
│   ├── contacts.yaml       # 【规划中】联系人分组、特殊人设绑定与规则库
│   └── knowledge/          # 【规划中】本地 RAG 知识文档库，Agent 会依据联系人读取相关文献
│
├── ui/                     # 🖥️ 图形化控制台界面 (Frontend)
│   ├── index.html          # 主页面（桌面宽屏仪表盘布局，暗色玻璃拟态主题）
│   ├── css/                # 样式文件目录
│   │   └── main.css        # 主样式表（333行，深色玻璃拟态主题、动画、组件样式）
│   └── js/                 # 脚本文件目录
│       └── app.js          # 主应用脚本（1294行，pywebview桥接、业务逻辑、UI交互）
│
├── main.py                 # 🚀 全局总管级入口脚本。负责调度 ui 或者直接运行 core/engine.py
├── requirements.txt        # 项目依赖库清单
├── ARCHITECTURE.md         # V3 核心逻辑的架构演进文档（记录红点驱动机制与防灾逻辑）
└── README_STRUCTURE.md     # 也就是本文档，整体项目的工程目录地图
```

## 为什么要进行拆分重构？
1. **彻底解耦**：未来启动 UI 图形界面后，界面可以随意崩溃、关闭，但只要把底层的 `core/engine.py` 放进独立线程/独立进程里运行，即使你关掉界面，底下的核心回复监控依旧岿然不动。
2. **纯粹的数据流动**：如果直接在代码里硬编码大模型提示词会非常混乱。未来将在 `data/contacts.yaml` 里通过可视化的选项去编辑配置，底层大脑直接通过 `yaml.load()` 读取。
3. **安全规范边界**：从脚本式开发向工程级产品跨越。各个 `core/*.py` 职责单一只做一件事。出了找坐标的问题去 `vision.py`，脑梗无法回复去 `agent.py`，点击点歪了去 `action.py`。
